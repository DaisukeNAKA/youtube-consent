# メール送信バックエンドの設定（Google Apps Script）── 運営者控えのみ送信モード

証票（PNG）を **運営者だけ** にメール送信するための、無料・サーバー不要のバックエンドです。
運営者ご自身のGmailから、運営者自身の宛先（`OWNER_EMAIL`）へ送信されます。

> **このモードの要点**
> - 証票メールは **運営者にのみ** 届きます。
> - **出演者・保護者/署名者には自動送信されません**（メールアドレスも取得しません）。
> - 宛先は `Code.gs` の `OWNER_EMAIL` に固定。クライアントからは変更できないため、
>   **任意の宛先へ送るメールリレー悪用ができない構造**です。
> - 出演者へ控えが必要な場合は、運営者が保存済みの証票を個別にお渡しください。

所要時間：約5分。**一度だけ**設定すれば、以降はアプリ側だけで完結します。

---

## 手順

### 1. プロジェクトを作る
1. https://script.google.com を開く（運営用のGoogleアカウントでログイン）
2. 左上「**新しいプロジェクト**」をクリック

### 2. コードを貼る
1. 最初から入っている `function myFunction() {}` を**全部消す**
2. このフォルダの [`Code.gs`](./Code.gs) の中身を**全部コピーして貼り付け**
3. 冒頭の設定を確認（既定で入っています）
   - `OWNER_EMAIL` … 証票の送信先（唯一の宛先。運営者のメール）
   - `SHARED_TOKEN` … アプリ側の合言葉と一致（**変更しないこと**。変える場合は index.html の `MAIL_TOKEN` も同じ値に）
   - `DAILY_CAP` … 1日の送信上限（既定90。撮影件数に合わせて小さくしてもよい）
4. 💾 保存（Ctrl/Cmd + S）

### 3. ウェブアプリとして公開
1. 右上「**デプロイ**」→「**新しいデプロイ**」
2. 歯車⚙「種類の選択」→「**ウェブアプリ**」
3. 設定：
   - 次のユーザーとして実行：**自分**
   - アクセスできるユーザー：**全員**（← 静的サイトから呼ぶために必要）
4. 「**デプロイ**」をクリック

### 4. 権限を承認（初回のみ）
1. 「アクセスを承認」→ 自分のGoogleアカウントを選択
2. 「このアプリは Google で確認されていません」と出たら
   → 「**詳細**」→「**（プロジェクト名）に移動**」→「**許可**」
   （自分で作ったスクリプトなので安全です）

### 5. URLをコピーしてアプリに登録
1. 表示された「**ウェブアプリのURL**」（`https://script.google.com/macros/s/……/exec`）をコピー
2. スマホで証票アプリを開く → ①画面の下「**⚙ メール送信の設定**」を開く
3. URL欄に貼り付け → 「**保存**」

これで完了です。⑤完了画面で証票が **運営者控えとして** 自動送信されます。

---

## 動作確認
- コピーしたURLをブラウザで開いて `{"ok":true,"status":"ready",...}` が表示されればOK。
- アプリで1件通してみて、**運営者のGmail** に証票PNG付きメールが届けば成功です（出演者側には届きません）。

## レスポンス仕様（フロントと整合）
- 送信成功: `{"ok":true,"status":"sent","message":"operator copy sent"}`
- 冪等（送信済み）: `{"ok":true,"status":"already_sent","message":"operator copy already sent"}`
- エラー: `{"ok":false,"code":"<コード>","message":"<説明>"}`
  （コード例: `unauthorized` / `no_image` / `bad_request` / `no_body` / `busy` / `daily_limit` / `quota_exceeded` / `server_error`）

## コードを更新したとき（重要）
Apps Scriptは、**「デプロイ」→「デプロイを管理」→ 鉛筆✏️→ バージョン「新バージョン」→ デプロイ** で再公開しないと反映されません。
**既存の `/exec` URL は通常そのまま使えます**（URLは変わりません）。

---

## セキュリティ / 注意

このウェブアプリは静的サイトから呼ぶために `アクセス：全員` が必須で、合言葉トークンも公開HTMLに含まれます。
つまり**トークンは“完全な秘密”にはできません**。しかし本モードでは、

- **宛先が `OWNER_EMAIL` に固定**され、クライアントから送られる宛先情報
  （`participantEmail` / `guardianEmail` / `recipients` / `to` / `cc` / `bcc` 等）は**完全に無視**されます。

そのため、**エンドポイントURLとトークンが露出しても、第三者は任意の宛先へ送れません**。
届くのは運営者本人のメールボックスのみで、メールリレー悪用リスクを大幅に下げています。

多重防御（すべて維持）:
- **合言葉（SHARED_TOKEN）**: 無差別なbotを弾く。
- **冪等化**: 同じ証票IDは二重送信しない（再送しても重複しません）。
- **日次上限（DAILY_CAP）＋ Gmail残枠チェック**: 万一乱用されても、あなたの通常メールが送れなくなる事態を防ぐ。
- **同時実行の直列化（LockService）**、**dataURL形式検証**、**添付ファイル名の安全化**、**code/message形式のエラー返却**。

**残る（限定的な）リスクと運用**:
- 第三者がトークンを知ると、あなたの受信箱に最大 `DAILY_CAP` 通/日のスパムを送り込める可能性はあります
  （＝あなた宛の迷惑メール止まり。他人には送れません）。気になる場合は下記を実施:
  1. `SHARED_TOKEN` を時々変更（`Code.gs` と `index.html` の `MAIL_TOKEN` を同じ新値にして両方を再デプロイ／再push）。
  2. `DAILY_CAP` を実運用の件数に合わせて小さくする（例: 20）。
  3. 本格運用が必要なら、外部メールAPI（Resend等）＋サーバ側秘密鍵＋認証付き構成へ移行。
- 送信元は運営者のGmail。Gmail無料枠の送信上限は約100通/日（Workspaceは約1,500通/日）。
