# メール送信バックエンドの設定（Google Apps Script）

証票（PNG）を **出演者本人・保護者・運営者** にメール送信するための、無料・サーバー不要のバックエンドです。
運営者ご自身のGmailから送信されます（1日あたり約100通まで送信可能）。

所要時間：約5分。**一度だけ**設定すれば、以降はアプリ側だけで完結します。

---

## 手順

### 1. プロジェクトを作る
1. https://script.google.com を開く（運営用のGoogleアカウントでログイン）
2. 左上「**新しいプロジェクト**」をクリック

### 2. コードを貼る
1. 最初から入っている `function myFunction() {}` を**全部消す**
2. このフォルダの [`Code.gs`](./Code.gs) の中身を**全部コピーして貼り付け**
3. 冒頭の設定を確認（既定で入っています）
   - `OPERATOR_EMAIL` … 運営者の控え宛先メール
   - `SHARED_TOKEN` … アプリ側の合言葉と一致（**変更しないこと**。変える場合は index.html の `MAIL_TOKEN` も同じ値に）
4. 💾 保存（Ctrl/Cmd + S）

### 3. ウェブアプリとして公開
1. 右上「**デプロイ**」→「**新しいデプロイ**」
2. 歯車⚙「種類の選択」→「**ウェブアプリ**」
3. 設定：
   - 次のユーザーとして実行：**自分**
   - アクセスできるユーザー：**全員**（← ここ重要）
4. 「**デプロイ**」をクリック

### 4. 権限を承認（初回のみ）
1. 「アクセスを承認」→ 自分のGoogleアカウントを選択
2. 「このアプリは Google で確認されていません」と出たら
   → 「**詳細**」→「**（プロジェクト名）に移動**」→「**許可**」
   （自分で作ったスクリプトなので安全です）

### 5. URLをコピーしてアプリに登録
1. 表示された「**ウェブアプリのURL**」（`https://script.google.com/macros/s/……/exec`）をコピー
2. スマホで証票アプリを開く → ①画面の下「**⚙ メール送信の設定**」を開く
3. URL欄に貼り付け → 「**保存**」

これで完了です。⑤完了画面で証票が自動的にメール送信されます。

---

## 動作確認
- コピーしたURLをブラウザで開いて `{"ok":true,"service":"consent-mailer"}` が表示されればOK。
- アプリで1件通してみて、運営者のGmailに証票PNG付きメールが届けば成功です。

## コードを更新したとき（重要）
Apps Scriptは、**「デプロイ」→「デプロイを管理」→ 鉛筆✏️→ バージョン「新バージョン」→ デプロイ** で再公開しないと反映されません（URLは変わりません）。

## セキュリティ / 注意（重要）

このウェブアプリは静的サイトから呼ぶために `アクセス：全員` が必須で、合言葉トークンも公開HTMLに含まれます。
つまり**トークンは“完全な秘密”にはできません**。そこで乱用の被害を抑えるため、`Code.gs` に多重防御を実装しています。

- **合言葉（SHARED_TOKEN）**: 無差別なbotを弾く。
- **運営者宛先の固定**: 運営者の控え先はサーバ側で固定。クライアントからは変更不可。
- **冪等化**: 同じ証票IDは二重送信しない（再送しても重複しません）。
- **日次上限（DAILY_CAP＝90通/日）＋ Gmail残枠チェック**: 万一乱用されても送信枠の枯渇（＝あなたの通常メールが送れなくなる事態）を防ぐ。
- **同時実行の直列化（LockService）**。

**残余リスク**: それでも「トークンを知る第三者が、あなたのGmail名義で最大90通/日を、入力したアドレス宛に送れる」可能性は残ります。個人チャンネルの目立たないURLなら現実的リスクは低いですが、ゼロではありません。

**もっと強固にしたい場合の選択肢**:
1. `SHARED_TOKEN` を時々変更する（`Code.gs` と `index.html` の `MAIL_TOKEN` を同じ新値にして両方を再デプロイ／再push）。
2. `DAILY_CAP` を実際の撮影件数に合わせて小さくする（例: 20）。
3. 出演者への自動送信をやめ、**運営者の控えのみ**に限定する（出演者へは運営者が後で手動転送）。宛先入力欄を無効化すれば実質リレーではなくなります。
4. 大量運用や本格運用が必要なら、外部メールAPI（Resend等）＋サーバ側秘密鍵＋認証付きの構成へ移行。

- 送信元は運営者のGmail。受信者にはあなたのGmailアドレスが表示されます。
- Gmail無料枠の送信上限は約100通/日です（Workspaceは約1,500通/日）。
