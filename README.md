# 出演許諾 証票作成アプリ

YouTube等の撮影現場で、一般出演者の出演許諾を電子的に回収するためのスマホ用Webアプリ（単一HTML・依存なし）。

**本番URL:** https://daisukenaka.github.io/youtube-consent/

## 現場フロー
1. **① 確認** — 同意文面を最後までスクロールして同意を解放。本人(18歳以上)／18歳未満の区分を選択。
2. **② 氏名** — 出演者名（未成年なら保護者名・続柄）。
3. **③ 撮影** — その場で撮影。撮影日時・GPS住所/座標を可視透かしで焼き込み。
4. **④ 署名** — 指で署名。
5. **⑤ 完了** — 「同意済」朱印付きの証票PNGを生成 → 保存／共有／**運営者控えメール送信**。

## メール送信（運営者控えのみ送信モード）
証票PNGは **運営者にのみ** メール送信されます。**出演者・保護者/署名者には自動送信されません**（メールアドレスも取得しません）。
送信先は Google Apps Script 側の `OWNER_EMAIL` に固定されており、クライアントからは変更できないため、
エンドポイントURL・トークンが露出しても任意の宛先へ送るメールリレー悪用はできません。

設定・仕組み・セキュリティの詳細 → [`apps-script/README.md`](apps-script/README.md)

## 構成
- `index.html` … フロントエンド（アプリ本体）。GitHub Pagesで配信。
- `apps-script/Code.gs` … メール送信バックエンド（Google Apps Script ウェブアプリ）。運営者が一度デプロイ。
- `apps-script/README.md` … バックエンドの設定手順とセキュリティ。

## 更新方法
- フロント（文面・UI）: `index.html` を編集して `git push` → GitHub Pagesに数十秒〜1分で自動反映。
- メール（`Code.gs`）: 編集後、Apps Scriptで「デプロイを管理 → 編集 → 新バージョン → デプロイ」で再公開（`/exec` URLは不変）。

## 注意
- カメラ・位置情報はHTTPS（本番URL）でのみ動作します。
